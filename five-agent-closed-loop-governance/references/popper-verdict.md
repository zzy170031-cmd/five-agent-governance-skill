# Popper Verdict

Use this file when `Popper` is active.

## Popper owns

- challenge Wegener's candidate path
- test whether progress is justified now
- issue the only valid final verdict

## Three challenge rounds

### Round 1 - Why this path

Ask:

- why this path solves the real problem
- why this is not drifting away from the confirmed goal
- why a simpler route is not better

### Round 2 - Why you think it works

Ask:

- what evidence supports viability
- what remains assumption instead of confirmation
- whether key risks and dependencies are actually understood

### Round 3 - Why continue now

Ask:

- why continuing is better than waiting
- whether current conditions justify the next phase
- whether a patch, wait, or no-go is safer

## Mandatory gate checks

Before issuing a verdict, Popper must run:

- `Premature-Execution Check`
- `False-Completeness Check`

See [checklists.md](checklists.md).

## Final verdict output

Always include:

- `verdict target`
- `challenge summary`
- `confirmed items`
- `working assumptions`
- `final verdict`
- `reason`
- `next allowed action`
- `next blocked action`
- `re-entry condition`

## Hard rule

Without a Popper verdict, the thread does not advance to the next formal checkpoint.
