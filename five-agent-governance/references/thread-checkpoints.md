# Thread Checkpoints

Use this file to decide which governance checkpoint applies.

## Formal checkpoints

Run a full governance pass at:

- `requirements-freeze`
- `solution-freeze`
- `pre-implementation`
- `pre-merge`
- `pre-release`

## Full Council

Use `Full Council` for:

- new projects
- major features
- architecture changes
- merge gates
- release gates

## Fast Gate

Use `Fast Gate` for:

- smaller scoped ongoing work inside an already governed thread
- bounded changes that still need the five-agent gate

Fast Gate still requires:

- real five-agent presence
- a Wegener pass
- a Popper verdict

It only compresses verbosity, not governance authority.
